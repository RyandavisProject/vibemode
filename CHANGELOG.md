# История изменений

Все заметные изменения проекта фиксируются здесь. Даты пишутся в формате `дд-мм-гггг`.

## 2.1 — 25-06-2026

### Исправлено

- Вернули время до сброса лимитов справа в строках `5ч` и `7д`.
- Для нового кабинета цифры лимитов остаются из API, а `Сброс через ...` берётся из видимого текста карточек.

### Проверки

- `scripts/check.ps1`: `120 tests OK`.

## 2.0 — 25-06-2026

### Добавлено

- Поддержан новый кабинет Vibemode: `https://portal.vibemod.pro/client`.
- Добавлено чтение лимитов через API кабинета:
  - `https://api.vibemod.pro/client/profile`
  - `https://api.vibemod.pro/client/usage/limits`
- Добавлен fallback-парсер видимого текста нового дашборда Vibemode.
- Добавлены тесты на новый API-формат, новый текстовый формат кабинета и UTF-8 рендер macOS-попапа.

### Изменено

- Остатки лимитов теперь считаются как `лимит - потрачено`:
  - 5 часов: `creditLimit5Hours - credits5Hours`
  - 7 дней: `creditLimit7Days - credits7Days`
- Версия синхронизирована между `pyproject.toml`, `__init__.py` и заголовком Windows-оверлея.
- README, privacy, security, publishing и architecture docs обновлены под Vibemode.
- Публичное название проекта исправлено на `Vibemode`; старые `vibemod`-алиасы оставлены для совместимости.
- macOS menu bar версия использует ту же общую логику чтения данных, что и Windows-оверлей.

### Исправлено

- Убрана подсказка на третьей строке `лимит/день`.
- Исправлена битая кодировка в macOS-попапе и macOS-диалогах.
- Исправлен публичный `CHANGELOG.md`, который был сохранён в повреждённой кодировке.

### Проверки

- `scripts/check.ps1`: `120 tests OK`.
- `scripts/run-once.ps1`: успешно прочитал новый кабинет Vibemode.

## 1.8.0 — 16-06-2026

### Добавлено

- macOS menu bar UI: иконка `NG ...` в системной строке меню и popover с лимитами.
- Нативный macOS NSPopover через PyObjC и локальный HTML-попап.
- macOS-скрипты установки, запуска, обновления и создания `.app`-ярлыка.

### Исправлено

- `run-overlay.sh` больше не должен убивать собственный процесс при поиске старых экземпляров.
- `.app`-ярлык устойчивее к перемещению папки проекта.

## 1.7.2 — 15-06-2026

### Исправлено

- Стабилизировано перетаскивание Windows-оверлея после долгой работы.
- Canvas-обработчики подсказок и строки `лимит/день` больше не пересоздаются на каждом render-цикле.
- Фоновое чтение лимитов ограничено очередью и timeout-защитами.

### Изменено

- Browser cache чистится только в безопасных cache-папках; cookies, Local Storage и Session Storage не удаляются.
- ZIP-обновление требует SHA256-проверку по умолчанию.

### Проверки

- `106 tests OK`.

## 1.7.1 — 15-06-2026

### Изменено

- `лимит/день` действует только в календарный день ручного задания.
- На следующий день третья строка скрывается, вчерашний лимит не переносится.
- Диалог нового дневного лимита предлагает расчётную цифру по формуле `остаток 7д / оставшееся время 7д`.
- Пункт `Сменить аккаунт` перенесён вниз, над `Закрыть`.

### Проверки

- `98 tests OK`.

## 1.7.0 — 14-06-2026

### Добавлено

- Режим `лимит/день` для ручного дневного бюджета расхода кредитов.
- Третья строка с расходом за день, заданным лимитом, процентом и прогресс-баром.
- Редактирование дневного лимита по двойному клику.
- Watchdog после сна Windows.
- SHA256-файл при сборке ZIP-релиза.

### Изменено

- UI больше не должен подвисать во время чтения сайта: работа с кабинетом вынесена в worker.
- ZIP updater использует allowlist известных файлов проекта.
- Последняя позиция окна сохраняется и восстанавливается надёжнее.

### Исправлено

- Кратковременные ошибки обновления не стирают последние успешные данные.
- Цвет дневного прогресса после 50% идёт по шкале: синий, жёлтый, оранжевый, красный.

### Проверки

- `98 tests OK`.

## 1.6.0 — 13-06-2026

### Добавлено

- Безопасный автоклик `Войти` для восстановления текущей сессии, если форма уже заполнена браузером и стабильна.
- Защита от второго одновременно запущенного экземпляра оверлея.

### Изменено

- `Сменить аккаунт` блокирует автоклик до первого успешного чтения нового аккаунта.
- Обновление лимитов стало мягче: при уже загруженном кабинете используется внутреннее обновление вместо полного reload.

### Исправлено

- После сна компьютера повторная форма входа снова открывает видимое окно ЛК.
- Убраны ложные периодические открытия окна входа.

## 1.5.2 — 12-06-2026

### Изменено

- При кратковременной проблеме обновления оверлей держит последние успешные данные и не показывает `нужен вход` сразу.
- Документация по приватности уточняет, что cookies, пароли и локальная сессия не попадают в GitHub/ZIP.

## 1.5.0 — 10-06-2026

### Добавлено

- Проверка новых версий через GitHub Releases при запуске и раз в сутки.
- Пункт меню обновления версии.
- Скрипт `scripts/update-and-restart.ps1` для Git- и ZIP-обновлений.
- Корневой установщик `Install-Vibemode.bat` для ZIP-установки без Codex, Claude Code и Git.
- Скрипт `scripts/package-release.ps1` для сборки ZIP-архива под GitHub Releases.

### Изменено

- Обновление не выполняется молча и останавливается при локальных изменениях в Git-установке.
