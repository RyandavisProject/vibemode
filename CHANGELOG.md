# История изменений

Все заметные изменения проекта фиксируются здесь. Даты пишутся в формате `дд-мм-гггг`.

## 2.5 — 03-07-2026

### Исправлено

- macOS popover теперь занимает всю ширину окна WebView, поэтому внутренние отступы слева и справа одинаковые.

### Проверки

- macOS menu bar/popover проверен на live-запуске после последних правок.
- Подтверждено чтение лимитов и времени сброса через Vibemode API.
- `tests.test_popover_server`: OK.
- `compileall src/neurogate_usage_overlay`: OK.

## 2.4 — 30-06-2026

### Добавлено

- Windows overlay получил настоящие прозрачные округлённые углы через Win32 color-key/window region.
- Добавлен controlled recovery после сна: hidden Playwright context пересоздаётся без удаления browser profile, cookies, state или history.
- Добавлено чтение реального времени сброса окон из `/client/me`: `window5HoursEndsAt` и `window7DaysEndsAt`.
- Добавлены отдельные модули для атомарного JSON state/history, Vibemode API parsing и Win32 window helpers.

### Изменено

- Прогресс-бары, верхняя строка и 1x/2x Windows layout стали стабильнее и аккуратнее.
- Дневной лимит больше не использует срок тарифа как замену времени сброса 7-дневного окна.
- Если сервер не отдаёт реальный reset time, overlay показывает пустое значение вместо придуманной цифры.

### Исправлено

- После сна overlay больше не должен залипать в состоянии `нужен вход`, если локальная браузерная сессия всё ещё валидна.
- Время сброса `5ч` и `7д` снова отображается справа в строках, но теперь берётся из серверных timestamp-полей.
- Убрана неверная автоподсказка лимита на день, основанная на `usage-records` или сроке подписки.

### Безопасность и надёжность

- Popover server получил лимит POST body и строгую token-защиту приватных action endpoint'ов.
- Debug/log paths не печатают raw portal text, cookies, token или приватные payload'ы.
- JSON state/history пишутся атомарно и устойчивее переживают corrupted/interrupted файлы.

### Проверки

- `scripts/check.ps1`: `172 tests OK`.
- `git diff --check`: чисто.
- `scripts/run-once.ps1`: успешно прочитал `5ч`/`7д` и реальные `reset_text` из Vibemode API.

## 2.3 — 27-06-2026

### Добавлено

- Добавлен `scripts/check-api-contract.py` — локальная диагностика будущего API-key режима.
- Скрипт проверяет `/client/me`, `/client/usage/limits`, `/client/api-keys/summary` и `/client/usage-records`.
- API-ключ вводится скрыто или через `VIBEMODE_API_KEY`; ключ не печатается и не сохраняется.
- Вывод диагностики показывает структуру ответа и полезные поля, но скрывает чувствительные поля вроде `token`, `secret`, `key`, `email`, `phone`, `name`, `login`.

### Изменено

- Windows/PC-оверлей получил дизайн-гипотезу ближе к macOS-попапу: мягкий тёмный фон, карточки строк и спокойные прогресс-бары.
- Контекстное меню, tooltip и окно `Лимит на день` приведены к единой тёмной палитре.
- В окне дневного лимита заголовок стал белым и жирным, поле ввода и кнопки получили скруглённые подложки, `OK` стал белым.
- Плашка интервала в шапке поднята, а между строками лимитов добавлены тонкие промежутки.

### Исправлено

- Убрано наложение текста в шапке при длинном статусе тарифа.
- Убрана тавтология вида `ост. 16дн осталось`: теперь остаётся компактное `ост. 16дн`.
- Геометрия строк `5ч`, `7д` и `лимит/день` разведена так, чтобы карточки не пересекались.

### Проверки

- `scripts/check.ps1`: `138 tests OK`.
- `git diff --check`: чисто.

## 2.2 — 27-06-2026

### Изменено

- macOS menu bar обновлён под Vibemode: белая иконка `VM`, компактный остаток `5ч` без подписи и формат `потрачено/лимит` при дневном лимите.
- Под цифрами в menu bar добавлена тонкая полоска прогресса: дневной лимит, если задан, иначе `5 часов`.
- Цвета всех полосок приведены к единой шкале: зелёный до 50%, жёлтый до 75%, красный после 75%.
- Правый клик по `VM …` открывает меню с лимитами и действиями.
- В macOS-попапе дневной лимит перенесён сразу под `7 дней`, добавлены тёмная тема, выбор интервала, `Перезапустить` и `Выход`.
- Вместо чекбокса `Не закрывать ЛК` используются действия `Показывать ЛК` / `Закрывать ЛК`.

### Исправлено

- Исправлена рекомендация дневного лимита: при остатке `7д` меньше суток предложение больше не превышает текущий остаток.
- Верхние строки лимитов снова показывают `остаток/общий лимит`, например `111.6M/120M`.
- Строка `лимит/день` выровнена по тем же координатам, что `5ч` и `7д`, и выводит дробь без пробелов вокруг `/`.
- Строка версии в Windows-меню отделена снизу разделителем; без доступного обновления она стала некликабельной информационной строкой.
- Windows-меню теперь ограничивается границами экрана и не открывается так, что его трудно закрыть или выбрать.
- macOS-попап теперь защищает локальные action-запросы token-проверкой.
- macOS ZIP-обновление требует SHA256 по умолчанию и умеет читать `.sha256` sidecar рядом с архивом.
- Исправлен macOS `.app`-ярлык: он запоминает реальную папку проекта и запускает правильный `scripts/run-overlay.sh`.
- Ожидание текста кабинета больше не использует длинный 45-секундный timeout на каждой polling-попытке, что снижает риск долгого старта при подвисшей странице.
- macOS `run-overlay.sh` больше не завершает процессы по широкому совпадению с названием бренда, а ищет точный запуск Python-модуля и Chrome с локальным profile path.
- Windows `run-overlay.ps1` тоже больше не завершает Python-процессы по широкому совпадению с названием бренда.
- GitHub Actions теперь включает macOS smoke/unit job с macOS-зависимостями.
- Release ZIP больше не включает внутренние `PROJECT_STATE.md`, `HANDOFF.md` и `security_best_practices_report.md`.
- `Закрывать ЛК` сразу закрывает Chrome-процессы локального профиля Vibemode и не ждёт долгий refresh.
- Убраны неподдерживаемые Chrome-флаги, которые вызывали предупреждения в браузере.
- Перед показом окна входа оверлей пробует auto-login в скрытом режиме, чтобы не открывать страницу логина лишний раз.
