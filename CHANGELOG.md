# История изменений

Все заметные изменения проекта фиксируются здесь.

Формат близок к [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/), версии
ведутся по смыслу: `major.minor.patch`.

## 1.7.0 — 14-06-2026

### Добавлено

- Режим `лимит/день`: пользователь может задать дневной лимит расхода кредитов
  из меню оверлея.
- Третья строка `лимит/день` с расходом за день, заданным лимитом, процентом
  достижения и прогресс-баром.
- Редактирование дневного лимита по двойному клику на строку.
- Watchdog после сна Windows: после длинной системной паузы оверлей сам
  запускает принудительное обновление.
- PID-файл `overlay.pid` для более точного управления предыдущим экземпляром.
- SHA256-файл при сборке ZIP-релиза.
- Передача release ZIP asset и checksum из GitHub Release API в updater.
- Комплексный audit report: `security_best_practices_report.md`.

### Изменено

- Старт стал быстрее визуально: чтение NeuroGate вынесено в отдельный worker,
  первый refresh запускается параллельно с UI.
- ZIP updater теперь обновляет allowlist известных файлов проекта, а не всё
  дерево установки.
- ZIP updater делает временный бэкап затронутых файлов и откатывает их при
  ошибке копирования.
- Сценарий запуска сначала пытается остановить конкретный процесс из
  `overlay.pid`, а fallback оставлен только для старых сборок.
- Последняя позиция окна сохраняется надёжнее и восстанавливается после
  перезапуска.
- `scripts/check.ps1` теперь проверяет синтаксис PowerShell-скриптов.
- Релизная упаковка теперь игнорирует временные `dist-*` папки, чтобы тестовые
  архивы не попадали внутрь публичного ZIP.

### Исправлено

- UI больше не должен подвисать во время долгого чтения страницы.
- Sync Playwright больше не вызывается из разных потоков.
- Debug-логи больше не сохраняют сырой текст ЛК.
- Логи ограничены по размеру и не разрастаются бесконечно.
- Усилен Windows single-instance lock для непустого lock-файла.
- Context-manager single-instance lock теперь явно сообщает об ошибке, если
  второй экземпляр пытается продолжить работу без полученного lock.
- Кратковременные ошибки обновления не стирают последние успешные данные.
- Цвет дневного прогресса после 50% теперь идёт по принятой шкале:
  синий до 50%, жёлтый после 50%, оранжевый около 75%, красный при 100% и выше.

### Проверки

- Проверки проекта: `96 tests OK`.
- `scripts/package-release.ps1` создаёт ZIP и `.sha256`.

## 1.6.0 — 13-06-2026

### Добавлено

- Безопасный автоклик `Войти` для обычного восстановления текущей сессии:
  оверлей нажимает кнопку только если форма входа уже заполнена браузером,
  остаётся стабильной несколько секунд и пользователь не меняет данные.
- Защита от второго экземпляра оверлея: повторный запуск не создаёт ещё одно окно
  и не начинает спорить за тот же локальный браузерный профиль.

### Изменено

- `Сменить аккаунт` теперь блокирует автоклик до первого успешного чтения нового
  аккаунта. Это позволяет спокойно ввести другую почту и пароль без возврата в
  старую сессию.
- Обновление лимитов стало мягче: если кабинет уже загружен, оверлей использует
  внутреннее обновление NeuroGate вместо полного reload страницы.
- Кратковременное появление формы входа больше не считается мгновенным
  разлогином: оверлей ждёт подтверждения, что вход действительно нужен.
- Скрипт запуска теперь закрывает старые экземпляры из ранних глобальных Python-запусков.

### Исправлено

- После сна компьютера повторная форма входа снова открывает видимое окно ЛК,
  а не оставляет оверлей застрявшим на `нужен вход`.
- Убраны ложные периодические открытия окна входа, которые появлялись из-за
  полного reload страницы и краткой промежуточной формы входа.

### Проверки

- Проверки проекта: `66 tests OK`.

## 1.5.2 — 12-06-2026

### Изменено

- Если во время обновления сайт кратковременно отдаёт `нужен вход` или `нет данных`,

  успешные данные на экране, перепроверяет сайт в фоне и показывает состояние
  входа только после подтверждённой серии неудачных попыток.

### Проверки

## 1.5.1 — 12-06-2026

### Добавлено

- Пункт меню `Сменить аккаунт`: он закрывает текущий браузерный контекст,
  удаляет локальный профиль оверлея и открывает чистое окно входа NeuroGate.

### Изменено

- Оверлей больше не нажимает кнопку входа автоматически, даже если форма уже

  на сайте NeuroGate.
- README и `docs/PRIVACY.md` уточняют, что cookies и сессия хранятся только
  локально в `%USERPROFILE%\.neurogate-usage-overlay\browser-profile` и не
  попадают в GitHub/ZIP-релиз.

### Проверки

- Проверки проекта: `51 tests OK`.

## 1.5.0 — 10-06-2026

### Добавлено

- Проверка новой версии при запуске и затем раз в сутки через публичные GitHub Releases.
- Пункт меню `Обновить до vX.Y.Z`, который появляется только при доступном обновлении.
- Скрипт `scripts/update-and-restart.ps1` для безопасного обновления и перезапуска.
- Корневой установщик `Install-NeuroGate-API.bat` для установки из ZIP без Codex,
  Claude Code и Git.
- Скрипт `scripts/package-release.ps1` для сборки ZIP-архива под GitHub Releases.

### Изменено

- Обновление программы отделено от обновления лимитов: пользователь сам запускает
  установку новой версии из меню.
- Git-установка обновляется через `git pull --ff-only`, а ZIP-установка скачивает
  ZIP новой версии с GitHub и раскладывает его поверх текущей папки.
- Автообновление Git-установки останавливается при локальных изменениях в папке
  проекта, чтобы не перезаписать пользовательские правки.

## 1.3.0 — 08-06-2026

### Добавлено

- Подсказки при наведении на оранжевые цифры лимитов.
- Для `5ч` показывается `Потрачено со сброса`.
- Для `7д` показывается `сегодня потрачено` или `сегодня потрачено с HH:MM`, чтобы пользователь видел точку
  отсчёта дневного расхода.
- Локальный файл `usage-daily.json` для расчёта дневного расхода по `7д`.
- Автонажатие кнопки входа, если видимая форма уже заполнена сохранёнными данными браузера.
  Позже это поведение удалено в `1.5.1`.

### Изменено

- Дневная точка отсчёта хранит только текущий день и перезаписывается при смене
  даты, поэтому файл не разрастается при длительном использовании.
