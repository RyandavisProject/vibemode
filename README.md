# NeuroGate API 1.7.1

Небольшой оверлей для лимитов NeuroGate API. Работает на **Windows** и **macOS**.

English note: this is a compact always-on-top overlay for NeuroGate API usage limits, supporting Windows and macOS.
The main documentation is in Russian because the primary users are Russian-speaking.

Оверлей читает страницу `https://portal.neurogate.space/client/usage` через
локальный профиль Chrome и показывает маленькое окно поверх рабочего стола:
название тарифа, остатки кредитов, время до сброса и интервал обновления.
После входа видимое окно Chrome обычно скрывается, а данные продолжают
обновляться из той же локальной браузерной сессии.

## Что показывает оверлей

Текущая страница лимитов NeuroGate отдаёт меньше данных, чем старая версия
интерфейса. Поэтому оверлей намеренно показывает только то, что реально видно
на странице сейчас:

- название тарифа;
- сколько времени осталось до окончания тарифа;
- остаток кредитов за 5 часов;
- остаток кредитов за 7 дней;
- полоски прогресса, совпадающие с полосками на сайте NeuroGate;
- подсказку при наведении на остаток `5ч`: сколько потрачено со сброса;
- подсказку при наведении на остаток `7д`: сколько потрачено сегодня с момента
  первой дневной точки отсчёта;
- время до сброса каждого лимита;
- опциональный дневной лимит расхода `лимит/день`, если пользователь задал его
  в меню;
- прогресс дневного лимита с предупреждающим цветом при приближении к лимиту;
- время последнего обновления;
- выбранный интервал обновления;
- уведомление о новой версии, если на GitHub опубликован свежий релиз.

Старые поля вроде общего количества токенов, кеша и пары `использовано / всего`
не показываются, если новая страница их больше не отдаёт в видимом интерфейсе.
Оверлей не подставляет сохранённые fallback-значения из прошлых запусков вместо
реальных данных. Но если на экране уже есть свежие лимиты текущей сессии,
кратковременный сбой обновления не стирает их сразу: оверлей оставляет последние
успешные данные, быстро перепроверяет сайт в фоне и показывает `нужен вход`
только после подтверждённой серии неудачных попыток.

При запуске оверлей держит один локальный экземпляр. Если случайно нажать ярлык
несколько раз, второй запуск не начнёт спорить за браузерную сессию и не будет
открывать лишние окна входа.

Подсказка `сегодня потрачено` считается локально по изменению остатка `7д`.
Если оверлей был установлен или запущен уже после полуночи, первый день может
считаться не с начала суток, а с момента первой свежей точки. В этом случае
подсказка честно показывает время: `сегодня потрачено с HH:MM`.

## Приватность

Проект работает локально на твоём компьютере.

- Оверлей не просит пароль NeuroGate в своём интерфейсе.
- Оверлей не собирает API-ключи.
- Оверлей не отправляет лимиты, cookies или данные аккаунта в этот проект,
  на отдельный сервер или в аналитику.
- Он читает только текст и элементы, уже видимые в твоей браузерной сессии.
- Cookies остаются на компьютере в локальном профиле Playwright/Chrome.
- После успешного входа видимое окно Chrome скрывается, а последующие чтения
  идут из той же локальной сессии.
- Если нужно войти в другой аккаунт, пункт меню `Сменить аккаунт` удаляет
  локальный профиль оверлея и открывает чистое окно входа NeuroGate.
- Автовход разрешён только для обычного восстановления текущей сессии: если
  форма входа уже заполнена браузером и несколько секунд не меняется, оверлей
  может сам нажать `Войти`. После `Сменить аккаунт` автоклик отключается до
  первого успешного входа в новый аккаунт.
- В меню по правой кнопке есть пункт `Не закрывать ЛК`. Если включить его,
  личный кабинет NeuroGate останется открытым отдельным окном Chrome. Если
  выключить, окно снова будет скрыто.
- В том же меню есть пункт `2x размер`: он пропорционально увеличивает оверлей
  в два раза и сохраняет выбранный размер между запусками.
- В меню можно задать `лимит/день`. Эта настройка хранится локально, действует
  только в текущий календарный день и на следующий день скрывается, чтобы
  пользователь вручную задал свежий лимит.
- Для подсказки `сегодня потрачено с HH:MM` хранится только текущая дневная точка отсчёта
  в локальном файле `usage-daily.json`; старые дни перезаписываются.
- Старые снимки лимитов не используются как запасные данные.

Основные локальные файлы:

```text
%USERPROFILE%\.neurogate-usage-overlay\browser-profile
%USERPROFILE%\.neurogate-usage-overlay\overlay-state.json
%USERPROFILE%\.neurogate-usage-overlay\usage-daily.json
```

Эту папку нельзя публиковать или передавать другим людям: в ней может быть
твоя локальная браузерная сессия. Файл `usage-daily.json` хранит только текущую
дневную точку отсчёта и перезаписывается при смене дня.

## Установка

### Windows — Вариант 1: обычная установка из ZIP

Этот способ подходит пользователю без Codex, Claude Code и Git.

1. Открой GitHub-страницу проекта:
   `https://github.com/RyandavisProject/neurogate-overlay`
2. Скачай ZIP-архив последней версии из раздела Releases.
3. Распакуй архив в обычную папку, например:
   `C:\NeuroGate API`
4. Дважды нажми файл:

```text
Install-NeuroGate-API.bat
```

Установщик создаст локальное Python-окружение, поставит зависимости и добавит
ярлык `NeuroGate API` на рабочий стол.

### Windows — Вариант 2: установка через Codex, Claude Code или Git

```powershell
git clone https://github.com/RyandavisProject/neurogate-overlay.git
cd neurogate-overlay
powershell -ExecutionPolicy Bypass -File .\scripts\install.ps1
```

Если проект уже скачан или склонирован, можно также запустить старый установщик
двойным кликом:

```text
scripts\install.bat
```

Установщик создаёт:

- локальное Python-окружение в `.venv/`;
- установку пакета в режиме разработки;
- ярлык на рабочем столе с названием `NeuroGate API`.

### macOS — установка

```bash
git clone https://github.com/RyandavisProject/neurogate-overlay.git
cd neurogate-overlay
bash scripts/install.sh
```

Или, если проект уже скачан:

```bash
bash scripts/install.sh
```

Установщик создаёт:

- локальное Python-окружение в `.venv/`;
- установку пакета в режиме разработки;
- `.app`-ярлык `NeuroGate API.app` в `~/Applications` (виден в Spotlight и Launchpad).

## Запуск

### Windows

Через PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\run-overlay.ps1
```

Или двойным кликом по ярлыку:

```text
NeuroGate API
```

### macOS

```bash
bash scripts/run-overlay.sh
```

Или открой `NeuroGate API.app` из `~/Applications` через Finder, Spotlight или Launchpad.

Первый запуск:

1. Оверлей пытается прочитать страницу лимитов в скрытом режиме.
2. Если нужен вход, откроется отдельное окно Chrome с локальным профилем.
3. Войди на сайте NeuroGate обычным способом.
4. Если форма входа уже заполнена браузером и несколько секунд не меняется,
   оверлей может сам нажать `Войти`, чтобы восстановить текущую сессию.
   Пароль при этом не сохраняется в проекте: проверяется только факт заполнения
   формы внутри локального Chrome.
5. После первого успешного чтения видимое окно Chrome будет скрыто.
6. Дальше обновления будут идти из той же локальной сессии.
7. Пока нужен вход, оверлей проверяет состояние каждые несколько секунд и
   быстро подхватывает успешную авторизацию.
8. После появления свежих данных автообновление идёт не чаще одного раза в
   минуту, если вручную не выбран другой интервал.

Если нужно держать личный кабинет открытым, нажми правой кнопкой по оверлею и
включи `Не закрывать ЛК`. Если выключить этот пункт, окно Chrome снова будет
скрыто. Настройка действует только во время текущего запуска: при следующем
обычном запуске приложение снова стартует в скрытом режиме, если не использовать
параметр `--show-browser`.

Если нужно сменить аккаунт NeuroGate, нажми правой кнопкой по оверлею и выбери
`Сменить аккаунт`. Оверлей удалит свой локальный браузерный профиль
`%USERPROFILE%\.neurogate-usage-overlay\browser-profile` и откроет чистое окно
входа. В этом режиме автоклик `Войти` отключён, поэтому можно спокойно ввести
другую почту и пароль. Это не трогает твой обычный Chrome-профиль.

## Управление

- Оверлей можно перетаскивать за любую видимую область.
- Последняя позиция окна сохраняется локально и восстанавливается при запуске.
- Левый клик по кнопке интервала, например `1м`, переключает интервалы.
- Выбранный интервал сохраняется и автоматически применяется при следующем запуске.
- Доступные интервалы: `1 мин`, `3 мин`, `5 мин`, `10 мин`, `15 мин`, `1 час`.
- Наведение на оранжевую цифру `5ч` показывает `Потрачено со сброса`.
- Наведение на оранжевую цифру `7д` показывает `сегодня потрачено` или
  `сегодня потрачено с HH:MM`, если день считается не с полуночи.
- Правый клик открывает компактное меню.
- Пункт `Сменить аккаунт` очищает локальную сессию оверлея и открывает чистый
  вход NeuroGate.
- Пункт `Не закрывать ЛК` держит страницу NeuroGate открытой отдельным окном.
- Пункт `2x размер` включает увеличенный интерфейс. Координаты и шрифты
  пересчитываются, поэтому оверлей не растягивается картинкой и остаётся чётким.
- Если на GitHub есть новая версия, в меню появляется пункт `Обновить до vX.Y.Z`.
- Повторный запуск ярлыка не создаёт второй оверлей поверх первого: активным
  остаётся один экземпляр приложения.
- `Esc` закрывает оверлей.
- `Ctrl+R` запускает ручное обновление с учётом минимальной паузы в 1 минуту.

## Обновления

Оверлей проверяет наличие новой версии при запуске и затем раз в сутки. Проверка
идёт через публичные GitHub Releases проекта и не требует логина, токена или
передачи приватных данных.

Если новая версия найдена, в меню по правой кнопке появится строка вроде:

```text
Доступна v1.7.1
Обновить до v1.7.1
```

После нажатия `Обновить до vX.Y.Z` запускается отдельный PowerShell-скрипт:

```text
scripts/update-and-restart.ps1
```

Для Git-установки скрипт скачивает свежий код через `git pull --ff-only`. Для
ZIP-установки он скачивает ZIP новой версии с GitHub и раскладывает его поверх
текущей папки, сохраняя `.venv` и локальные пользовательские данные вне проекта.
После этого обновляется локальная установка пакета, пересоздаётся ярлык и
оверлей запускается заново.

Для ZIP-релизов скрипт сборки создаёт рядом с архивом файл `.sha256`. Если
GitHub Release содержит ZIP-asset проекта и checksum, updater проверяет архив
перед распаковкой. ZIP-обновление раскладывает только известные файлы проекта и
может откатить затронутые файлы из временного бэкапа, если копирование сорвалось.

Если Git-установка содержит локальные изменения в папке проекта, обновление
останавливается, чтобы ничего не перезаписать.

## История изменений

### 1.7.1 — 15-06-2026

- Дневной лимит теперь действует только в календарный день ручного задания:
  на следующий день строка `лимит/день` скрывается и вчерашний лимит не
  переносится.
- При новом задании дневного лимита диалог предлагает расчётную цифру по формуле
  `остаток 7д / оставшееся время 7д`, где часы переводятся в десятичную часть
  дня. Значение сохраняется только после ручного подтверждения.
- Пункт меню `Сменить аккаунт` перенесён вниз, прямо над `Закрыть`.
- Проверки проекта: `98 tests OK`.

### 1.7.0 — 14-06-2026

- Добавлен режим `лимит/день`: в меню можно задать дневной лимит расхода
  кредитов, после чего в оверлее появляется третья строка с расходом за день,
  заданным лимитом, процентом и прогресс-баром.
- Цвет дневного прогресса предупреждает о перерасходе: до 50% обычный синий,
  после 50% жёлтый, к 75% оранжевый, при 100% и выше красный.
- Исправлен warning-цвет дневного прогресса: на значениях вроде 53% полоса уже
  заметно жёлтая, а не визуально синяя.
- По двойному клику на строку `лимит/день` открывается редактирование лимита.
- UI не должен подвисать во время чтения NeuroGate: работа с сайтом вынесена в
  отдельный worker-поток.
- Добавлен watchdog после сна Windows: если компьютер проснулся после длинной
  паузы, оверлей принудительно перепроверяет лимиты, не стирая старые успешные
  данные при кратковременной ошибке.
- Последняя позиция окна сохраняется надёжнее и восстанавливается после
  перезапуска.
- Debug-логи больше не сохраняют сырой текст ЛК, а размер логов ограничен.
- ZIP-релиз собирается с `.sha256`, update checker передаёт updater ZIP-asset
  релиза и checksum, а ZIP-обновление использует allowlist и временный rollback.
- Релизная упаковка защищена от случайного включения временных `dist-*` папок
  и тестовых архивов.
- Усилен single-instance lock: вспомогательный context-manager теперь явно
  падает, если lock уже занят.
- Обновлён второй скриншот README с актуальным меню и дневным лимитом.
- Проверки проекта: `98 tests OK`.

### 1.6.0 — 13-06-2026

- Возвращён безопасный автоклик `Войти`: он работает только если форма входа

## Ограничения

- Парсер читает видимый текст и элементы страницы. Если NeuroGate изменит
  подписи или структуру интерфейса, проекту может понадобиться небольшое
  обновление.
- Приложение ориентировано на Windows, потому что использует поведение Tkinter
  для десктопного окна и Windows-скрипты для ярлыка.
- Оверлей не обходит вход и не продлевает сессию сайта. Если NeuroGate разлогинит
  пользователя, откроется видимое окно Chrome для повторного входа. Если форма
  уже заполнена сохранённой браузерной сессией и не меняется, оверлей может
  нажать `Войти`; при смене аккаунта автоклик отключён. После успешного чтения
  окно снова скрывается.
- Оверлей не показывает сохранённые fallback-значения из прошлых запусков.
- При кратковременной проблеме обновления оверлей держит последние успешные
  данные текущей сессии на экране и не заменяет их сразу на `нужен вход`.
- Расход `сегодня потрачено` становится точным с полуночи только тогда, когда
  оверлей уже видел дневную точку отсчёта. В первый день после установки он
  может показывать расход с момента первого свежего чтения и явно подписывает
  это временем.
