# Vibemode 2.2

Небольшой оверлей для лимитов Vibemode API. Работает на **Windows** и **macOS**.

English note: this is a compact overlay for Vibemode API usage limits, supporting Windows and macOS.
The main documentation is in Russian because the primary users are Russian-speaking.

На **Windows** оверлей отображается как маленькое окно поверх рабочего стола.
На **macOS** оверлей живёт в системном menu bar — кликаешь на белую иконку `VM` и видишь
выпадающий попап с прогресс-барами по каждому окну лимитов.

Оверлей читает кабинет `https://portal.vibemod.pro/client` через
локальный профиль Chrome и API `https://api.vibemod.pro`, затем показывает:
название тарифа, остатки кредитов, прогресс-бары и интервал обновления.
После входа видимое окно Chrome скрывается, а данные продолжают обновляться
из той же локальной браузерной сессии.

## Что нового в 2.2

- macOS-версия обновлена под Vibemode: белая иконка `VM`, цифра лимита и тонкая полоска прогресса в menu bar.
- Если дневной лимит не задан, в menu bar показывается остаток лимита `5 часов`; если задан — `потрачено/лимит`.
- В macOS-попапе добавлены тёмная тема, дневной лимит после `7 дней`, выбор интервала, обновление, перезапуск и выход.
- Все полоски используют одну шкалу: зелёная до 50%, жёлтая до 75%, красная после 75%.
- `Показывать ЛК` / `Закрывать ЛК` заменили старый чекбокс `Не закрывать ЛК`.
- Убраны Chrome-предупреждения от неподдерживаемых флагов.

Полная история версий — в разделе [История изменений](#история-изменений) и в [CHANGELOG.md](CHANGELOG.md).

## Что показывает оверлей

- название тарифа и время до окончания подписки;
- остатки кредитов за **5 часов** и **7 дней**;
- прогресс-бары по данным Vibemode API;
- время до сброса каждого лимита;
- *(Windows)* подсказку при наведении на остаток `5ч` — сколько потрачено со сброса;
- *(Windows)* подсказку при наведении на остаток `7д` — сколько потрачено сегодня;
- опциональный **дневной лимит расхода** — задаётся вручную, показывает потрачено/лимит/процент;
- время последнего обновления;
- уведомление о новой версии, если опубликован свежий релиз на GitHub.

## Приватность

Проект работает локально на твоём компьютере.

- Оверлей не просит пароль Vibemode в своём интерфейсе.
- Оверлей не собирает API-ключи.
- Оверлей не отправляет лимиты, cookies или данные аккаунта в этот проект,
  на отдельный сервер или в аналитику.
- Он читает только текст и элементы, уже видимые в твоей браузерной сессии.
- Cookies остаются на компьютере в локальном профиле Playwright/Chrome.
- После успешного входа видимое окно Chrome скрывается.
- Если нужно войти в другой аккаунт — пункт `Сменить аккаунт` удаляет
  локальный профиль оверлея и открывает чистое окно входа Vibemode.

Основные локальные файлы:

```text
~/.neurogate-usage-overlay/browser-profile
~/.neurogate-usage-overlay/overlay-state.json
~/.neurogate-usage-overlay/usage-daily.json
```

Эту папку нельзя публиковать или передавать другим людям: в ней может быть
твоя локальная браузерная сессия.

## Требования

### Windows

- Windows 10/11
- Python 3.10+
- Google Chrome
- доступ к сайту Vibemode

### macOS

- macOS 13 Ventura или новее
- Python 3.10+ (рекомендуется через [Homebrew](https://brew.sh): `brew install python`)
- доступ к сайту Vibemode

> На macOS используется встроенный WebKit (WKWebView) через PyObjC — дополнительный
> браузер не нужен. При первом запуске Playwright скачает Chromium автоматически.

## Установка

### Windows — Вариант 1: обычная установка из ZIP

Этот способ подходит пользователю без Codex, Claude Code и Git.

1. Открой GitHub-страницу проекта:
   `https://github.com/RyandavisProject/vibemode`
2. Скачай ZIP-архив последней версии из раздела Releases.
3. Распакуй архив в обычную папку, например:
   `C:\Vibemode`
4. Дважды нажми файл:

```text
Install-Vibemode.bat
```

Установщик создаст локальное Python-окружение, поставит зависимости и добавит
ярлык `Vibemode` на рабочий стол.

### Windows — Вариант 2: установка через Codex, Claude Code или Git

```powershell
git clone https://github.com/RyandavisProject/vibemode.git
cd vibemode
powershell -ExecutionPolicy Bypass -File .\scripts\install.ps1
```

### macOS — установка

```bash
git clone https://github.com/RyandavisProject/vibemode.git
cd vibemode
bash scripts/install.sh
```

Или, если проект уже скачан:

```bash
bash scripts/install.sh
```

Установщик:
- создаёт локальное Python-окружение в `.venv/`;
- ставит зависимости включая PyObjC для нативного UI;
- создаёт `.app`-ярлык `Vibemode.app` в `~/Applications` (виден в Spotlight и Launchpad).

## Запуск

### Windows

Через ярлык `Vibemode` на рабочем столе или через PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\run-overlay.ps1
```

### macOS

Открой `Vibemode.app` из `~/Applications` через Finder, Spotlight или Launchpad.

Или из терминала:

```bash
bash scripts/run-overlay.sh
```

**Первый запуск:**

1. Оверлей пытается прочитать страницу лимитов в скрытом режиме.
2. Если нужен вход, откроется отдельное окно Chromium с локальным профилем.
3. Войди на сайте Vibemode обычным способом.
4. После успешного входа окно Chromium скроется.
5. В menu bar появится белая иконка `VM` и актуальный остаток лимита `5ч`.
6. Дальше обновления идут автоматически раз в 1 минуту (по умолчанию).

## Управление (macOS)

Кликни на иконку `VM …` в menu bar — откроется попап с данными по лимитам.
Если дневной лимит не задан, рядом с иконкой показывается только остаток `5ч`
например `102.2M`. Если дневной лимит задан, показывается расход за день и лимит
через дробь, например `14.8M/87.6M`.
Тонкая полоска под цифрой показывает прогресс дневного лимита, а если дневной
лимит не задан — прогресс лимита `5 часов`. Длина полоски равна ширине цифры:
зелёная до 50%, жёлтая до 75%, красная после 75%.
Правый клик по `VM …` открывает то же меню с лимитами и действиями.

**Содержимое попапа:**

- Имя аккаунта и время до окончания подписки
- По каждому окну лимитов: прогресс-бар, процент использования, остаток и время до сброса
- **Лимит на день** сразу после блока `7 дней` (если задан): прогресс-бар с потраченным/лимитом за текущий день
- Кнопка `⊕ Задать лимит на день` — нажимаешь, вводишь лимит (например `56M`);
  программа сама предложит расчётную цифру из остатка `7д ÷ дни до сброса`
- Кнопка `Обновить` — принудительное обновление данных
- `Интервал обновления: 1 мин` — нажимаешь для переключения между `1м / 3м / 5м / 10м / 15м / 1ч`
- `Показывать ЛК` / `Закрывать ЛК` — открыть или сразу закрыть окно личного кабинета
- `Перезапустить` — закрыть текущий процесс и поднять свежую версию
- `Выход` — завершить приложение

**Дневной лимит:**

Помогает равномерно расходовать 7-дневный лимит. Задаётся вручную один раз в день.
Программа предлагает рекомендуемое значение по формуле:

```
рекомендуемый лимит/день = остаток 7д ÷ дней до сброса
```

Когда потраченное за день превышает заданный лимит — прогресс-бар становится красным.
Нажатие на карточку дневного лимита сбрасывает его.
Лимит автоматически сбрасывается в следующий календарный день.

## Управление (Windows)

- Оверлей можно перетаскивать за любую видимую область.
- Левый клик по кнопке интервала переключает интервалы обновления.
- Наведение на оранжевую цифру `5ч` показывает `Потрачено со сброса`.
- Наведение на оранжевую цифру `7д` показывает `сегодня потрачено`.
- Правый клик открывает компактное меню.
- Пункт `Не закрывать ЛК` держит страницу Vibemode открытой отдельным окном.
- Пункт `2x размер` включает увеличенный интерфейс.
- Пункт `лимит/день` — задать дневной лимит расхода кредитов.
- Строка версии показывает `v.2.2 (последняя)` или `v.2.2 (доступна v.2.3)`.
- `Esc` закрывает оверлей.
- `Ctrl+R` запускает ручное обновление.

## Обновления

Оверлей проверяет наличие новой версии при запуске и раз в сутки через публичные
GitHub Releases.

Если установлена последняя версия, меню показывает `v.2.2 (последняя)`.
Если доступно обновление, эта же строка меняется на `v.2.2 (доступна v.2.3)`
и становится единственной кнопкой обновления.

**Windows:** в меню появляется строка `v.2.2 (доступна v.2.3)` → нажимаешь →
скрипт скачивает и применяет обновление автоматически.

**macOS:** в попапе появляется строка `v.2.2 (доступна v.2.3)` → нажимаешь → запускается
`scripts/update-and-restart.sh`.
